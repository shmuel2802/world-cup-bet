# 🏆 Mundial Bet - האפליקציה החברתית להימורי מונדיאל

אפליקציית **Full-Stack** מרהיבה, מעוצבת ברמה הגבוהה ביותר (Obsidian Stadium Theme), המיועדת להתערבויות והימורים חברתיים על משחקי המונדיאל בין חברים באמצעות מטבעות וירטואליים במשחק!

הפרויקט כולל שרת **Node.js + Express** עצמאי, ממשק משתמש מתקדם ב-**React.js + Vite**, אפליקציית **Progressive Web App (PWA)** מותקנת למובייל, ופרויקט **Android Native** מובנה באמצעות **Ionic Capacitor** המאפשר ליצור קובץ התקנה (APK) לטלפונים ניידים.

---

## 🚀 תכונות עיקריות

*   **ממשק משתמש מעוצב מטורף (Premium Theme):** סגנון אצטדיון כהה, שילובי צבעים של ירוק אזמרגד ניאון (Electric Emerald) וזהב גביע (Championship Gold), אפקטים של זכוכית (Glassmorphism), ואנימציות ריחוף ודופק חלקות במיוחד.
*   **הרשמה חופשית:** כל שחקן יכול להירשם בקלות, להצטרף לתחרות ולקבל יתרת פתיחה של **1,000 מטבעות**.
*   **מכניקת הימורים כפולה (מכפילים אטרקטיביים):**
    *   **הימור תוצאה (HOME/DRAW/AWAY):** מכפיל קבוע של **פי 2** על סכום ההימור.
    *   **הימור תוצאה מדויקת (Exact Score):** מכפיל עוצמתי של **פי 5** במידה והתוצאה קולעת בדיוק!
*   **מערכת סנכרון משחקים ותוצאות אוטומטית:** חיבור ל-API חיצוני למשיכת זמני משחקים ותוצאות בזמן אמת, כולל מערכת סימולציית משחקים חכמה (Simulation Mode) לצרכי בדיקות והרצה מקומית.
*   **פאנל ניהול מאובטח (Admin Dashboard):** כניסת מנהל ייעודית המאפשרת לעדכן תוצאות ידנית, לקבוע סטטוס משחקים, לחשב הימורים באופן מיידי, ולשלוט בהגדרות המערכת.
*   **טבלת מובילים (Leaderboard):** דירוג שחקנים דינמי לפי יתרה, אחוזי הצלחה ומספר הימורים כולל, המציג תגי כבוד מעוצבים למובילים.
*   **סנכרון ענן דינמי:** התממשקות מובנית למסד נתונים בענן (כמו Firebase Realtime Database) דרך REST API פשוט - מבטיח שגם בשרתים חינמיים ללא דיסק קשיח קבוע (כמו Render), הנתונים שלכם לא יימחקו לעולם!

## 💻 הרצה מקומית במחשב (Local Setup)

אם ברצונך להפעיל את האפליקציה במחשב האישי שלך למטרות בדיקה ופיתוח:

### 1. הפעלת השרת (Backend)
1. פתח טרמינל, נווט לתיקיית `backend`:
   ```bash
   cd backend
   npm install
   ```
2. הפעל את השרת:
   ```bash
   node server.js
   ```
   *השרת יפעל בכתובת `http://localhost:5000` ויאתחל קובץ `db.json` מקומי.*

### 2. הפעלת ממשק המשתמש (Frontend)
1. פתח טרמינל נוסף, נווט לתיקיית `frontend`:
   ```bash
   cd frontend
   npm install
   ```
2. הפעל את שרת הפיתוח:
   ```bash
   npm run dev
   ```
   *ממשק המשתמש יפתח בדפדפן בכתובת `http://localhost:5173`.*

## ☁️ העלאה לאינטרנט ופריסה בענן (Cloud Deployment)

על מנת שתוכל לשחק עם החברים מכל מקום ומכל מכשיר נייד (מבלי שהמחשב שלך יצטרך לפעול כשרת), נבצע פריסה מלאה לאינטרנט.

### שלב א': יצירת מסד נתונים בענן (חינם)
מכיוון ששרתי ענן חינמיים מוחקים את הקבצים המקומיים שלהם בכל הפעלה מחדש (Ephemeral Disk), יצרנו מנגנון שמגבה ושולף את הנתונים ישירות ממסד נתונים של **Firebase Realtime Database**.

1. כנס ל-[Firebase Console](https://console.firebase.google.com/) והירשם בחינם.
2. צור פרויקט חדש (למשל `world-cup-bet`).
3. בתפריט הצדדי, תחת **Build**, בחר ב-**Realtime Database** ולחץ על **Create Database**.
4. בחר מיקום לשרת (למשל אירופה), ובשלב של חוקי האבטחה בחר ב-**Start in test mode** (כדי לאפשר קריאה וכתיבה ללא חסימות).
5. תקבל כתובת URL ייחודית, לדוגמה: `https://world-cup-bet-default-rtdb.europe-west1.firebasedatabase.app/`.
6. **חשוב מאוד:** לצורך התקשורת של השרת, נוסיף את הסיומת `data.json` לכתובת. הכתובת הסופית שתצטרך תיראה כך:
   `https://world-cup-bet-default-rtdb.europe-west1.firebasedatabase.app/data.json`

### שלב ב': העלאת הקוד ל-GitHub שלך
כדי להעלות את הקוד לחשבון הגיטהאב שלך (`shmuel2802`):

1. כנס לאתר [GitHub](https://github.com/) והתחבר לחשבונך.
2. צור מאגר חדש (Repository) בשם: **`world-cup-bet`** (השאר אותו ציבורי ואל תוסיף README או gitignore חדשים).
3. פתח טרמינל במחשב בתיקיית השורש של הפרויקט (`world-cup-bet`) והרץ את הפקודות הבאות כדי לקשר ולדחוף את הקוד:
   ```bash
   git remote add origin https://github.com/shmuel2802/world-cup-bet.git
   git branch -M main
   git push -u origin main
   ```

### שלב ג': פריסת השרת ב-Render.com (חינם)
1. כנס לאתר [Render](https://render.com/) והרשם באמצעות משתמש ה-GitHub שלך.
2. לחץ על **New +** ובחר ב-**Web Service**.
3. חבר את חשבון הגיטהאב שלך ובחר במאגר **`world-cup-bet`**.
4. הגדר את ההגדרות הבאות:
   *   **Name:** `world-cup-bet-server`
   *   **Root Directory:** `backend` (זה קריטי! השרת נמצא בתיקיית המשנה)
   *   **Runtime:** `Node`
   *   **Build Command:** `npm install`
   *   **Start Command:** `node server.js`
5. נווט ללשונית **Environment** (משתני סביבה) והוסף את המשתנים הבאים:
   *   **Key:** `CLOUD_DB_URL`
   *   **Value:** הכתובת המלאה שקיבלת מ-Firebase (כולל הסיומת `/data.json`).
   *   **Key:** `FOOTBALL_API_KEY`
   *   **Value:** מפתח API תקין של [Football-Data.org](https://www.football-data.org/) כדי למשוך תמיד את כל משחקי המונדיאל האמיתיים (כולל שלבי נוקאאוט עתידיים כשהם מתפרסמים).
6. לחץ על **Deploy Web Service**. השרת שלך יעלה לאוויר ותקבל כתובת אינטרנט ציבורית מובטחת (למשל `https://world-cup-bet-server.onrender.com`).

## 📱 עדכון ממשק הלקוח והאפליקציה למובייל

לאחר שהשרת שלך רץ בענן, עליך לעדכן את האפליקציה כדי שתתחבר לשרת החדש באינטרנט במקום למחשב המקומי (`localhost`).

### שלב א': עדכון הכתובת בקוד הלקוח
1. פתח את הקובץ `frontend/src/App.jsx`.
2. בשורות הראשונות של הקובץ, מצא את השורה:
   ```javascript
   const API_URL = 'http://localhost:5000/api';
   ```
3. החלף אותה בכתובת השרת הציבורי שלך מ-Render, למשל:
   ```javascript
   const API_URL = 'https://world-cup-bet-server.onrender.com/api';
   ```

### שלב ב': בנייה מחדש וסנכרון לאנדרואיד
לאחר שינוי הכתובת, יש לבצע בנייה (Build) של ממשק המשתמש ולסנכרן את הקבצים החדשים אל פרויקט האנדרואיד המובנה:
1. פתח טרמינל בתיקיית `frontend`:
   ```bash
   cd frontend
   npm run build
   npx cap sync
   ```

### שלב ג': קימפול קובץ APK סופי ומעודכן
הגדרנו עבורך סקריפט קימפול אוטומטי. לאחר ביצוע הסנכרון, תוכל לייצר קובץ APK חדש שיותקן בטלפונים ויתחבר ישירות לשרת הענן שלך!
הפעל את פקודת הקימפול בתוך תיקיית האנדרואיד, או פתח את פרויקט האנדרואיד הממוקם ב-`frontend/android` בתוך **Android Studio** ובחר:
`Build -> Build Bundle(s) / APK(s) -> Build APK(s)`.

קובץ ה-APK המעודכן יחכה לך מוכן בנתיב:
`frontend/android/app/build/outputs/apk/debug/app-debug.apk`

שלח את ה-APK המעודכן לחברים שלך בווטסאפ או במייל, התקינו ותתחילו להמר! 🏆⚽🔥
